# CV tailoring — best practices (the doctrine)

Distilled from research across four lenses (Aakash Gupta's PM-resume corpus, authoritative
career-services + recruiter guidance, the over-tailoring/authenticity literature, and
executive-search/layout-integrity sources). This is the rulebook the skill follows. Alex's
target level is **VP Product / CPO**, so executive nuances dominate.

## Core principles (ranked, non-negotiable)

1. **Re-prioritize TRUE facts only; never fabricate.** Tailoring = reordering, re-lensing, and
   keyword-mirroring of real accomplishments — never new claims. Every line must pass the
   **cold-defense test**: *"Could Alex defend this in a detailed interview without inventing
   context?"* If no → cut or downgrade.
2. **At exec level, POSITION — don't keyword-match.** What survives an exec-recruiter's read is a
   consistent, quantified leadership narrative (scope, P&L, org size, transformation), not JD
   keyword coverage. Over-sanding Alex's voice to mirror a JD is a *bigger* threat to a callback
   than a missing buzzword.
3. **Over-tailoring is the dominant risk, not under-tailoring.** A CV that echoes the JD
   near-verbatim or reads as AI-generated lands in the "polished but empty / over-fit / fake
   candidate" pile. Surgical edits beat rewrites. **It is completely fine to change little or
   nothing if the base already fits.**
4. **Edit length-neutral, always.** Match the *rendered line count* of any swapped text to the
   original. One longer line cascades into reflow, page-count growth, and widows. (Enforced
   mechanically — see SKILL.md verification gate.)
5. **Front-load relevance.** The first scan hits name, title, scope, dates, and the top of each
   section. JD-relevant scope/outcomes must sit in the scan path.
6. **Bullets lead with quantified impact** (X-Y-Z: "Accomplished X, measured by Y, by doing Z").
   For Alex, metrics must be org/P&L/strategy-scale (ARR, org size, portfolio breadth), **not**
   IC-PM scale (activation %, app rating).
7. **~75-80% JD-term match, woven in context — never 100%, never a keyword dump.** 80% clears
   filters; higher tips into stuffing that both NLP-ATS and humans penalize.
8. **One strong base, surgical per-job edits.** Rewriting from scratch per job manufactures
   AI-sameness and version drift (two versions describing different "people" = a red flag).

## What to tailor & where — and the safe degree of change

| Zone | What to do | Safe degree |
|---|---|---|
| **Headline / title line** (under the name) | Nudge toward the role's title where TRUE & natural (Alex's line already lists CPO / VP Product / Director PM / Head of Product). | Reorder/emphasize within the existing list. Low risk, high value. Don't invent a title. |
| **About / summary** | Re-point emphasis to THIS role's mandate (scale vs. 0→1 vs. platform vs. agentic-AI vs. GM/P&L). Seed 1-3 top JD terms naturally. | Re-lens prose; keep the through-line and voice. Length-neutral. |
| **Skills list** (sidebar) | Reorder so JD-named skills lead; swap a listed term for the JD's exact synonym **only if Alex genuinely has it**. | Reorder freely. Swap term-for-term length-neutrally. Never add a skill Alex can't demonstrate; never grow the block's line count. |
| **Industry / Sweet-spots ◆-tags** | Reorder so the JD's domain leads; swap one tag for the JD's word for the same true thing. | Reorder freely (zero length change). Term swaps must be ~equal width. |
| **Experience bullets** | Make the most JD-relevant TRUE accomplishment lead each role; re-phrase the same fact in the employer's vocabulary. | Reorder & re-phrase freely; keep each line's wrap identical. Never add an accomplishment that didn't happen. |

**Cross-zone:** spread keywords across summary + skills + bullets naturally. The JD is not a
100%-inclusion checklist.

### The safe-levers checklist — run through ALL of these every time
Restraint means *not forcing risky edits* — it does **not** mean skipping the free ones. Under-using
a zero-risk lever is the most common way these runs miss a 10. For every JD, explicitly decide
each of the following (take it, or record a concrete reason not to):

1. **Headline title line** — if the JD's role title (or its nearest equivalent) is already in Alex's
   list, is it positioned to be seen? Reordering it forward is **zero-length-change, zero-honesty-risk**
   — the single safest edit available. Take it unless leading with it would *under*-position Alex
   (e.g. fronting "Director PM" ahead of "CPO" for a role below his ceiling — then leave as-is and say so).
2. **Skills list** — reorder so the JD's named skills lead their group. Zero line-count change.
3. **Industry + Sweet-spots ◆-tags** — reorder so the JD's domain/mandate leads. Near-zero risk.
4. **One named-true JD requirement that's currently absent** — if the JD names a requirement Alex
   genuinely meets but the word never appears (e.g. "packaging", "KPI"/"analytics"), can ONE
   length-neutral word-swap inside an existing bullet surface it (e.g. "pricing" → "pricing &
   packaging")? Attempt it; keep only if the line-wrap holds. This is how you mirror a named
   requirement without stuffing.
5. **About / summary re-point** — does the role's core mandate sit in the scan path (the summary's
   lead clauses or the first experience block)? If the strongest match is buried below the fold,
   attempt a **length-neutral** re-lens of the summary (see the technique in `cv-inventory.md`). Keep
   it only if the wrap holds and it reads in Alex's voice without echoing the JD; otherwise leave it.

The bar for a 9-10 on JD-alignment: every lever above was either taken or has a concrete,
defensible reason it was skipped — AND the role's mandate sits somewhere in the scan path.

## Hard limits / red lines

- **No fabrication** — no invented metrics, no claimed scope Alex didn't own, no skills not held.
- **No ownership/proficiency inflation** — "Led" only where led; "Expert" only where provable;
  distinguish owned vs. influenced.
- **No voice erasure** — preserve Alex's distinctive leadership narrative and natural phrasing.
- **No version drift** — every tailored version is recognizably the same person as the base.
- **Honesty is a hard gate** (rubric dim. E must be 10).

### Signals that read as forced/gamed (avoid all)
- Near-verbatim JD echo (the clearest over-tailoring tell).
- Keyword stuffing: raw lists, a term repeated unnaturally, oversized sections.
- AI-sameness: "polished but empty" bullets, uniform line rhythm, metrics without context.
- Awkward phrasing crammed to fit a term — reads as inexperience/desperation. **Absence of a
  buzzword beats a gamed sentence.**

### When to deliberately change nothing
The JD wording would force unnatural phrasing; the existing line already covers the requirement
in Alex's words; or the change would dilute the leadership through-line.

## Executive (VP/CPO) nuances
- Lead with scope + quantified transformation, not operational duties.
- Keep P&L scope, team size, current title in plain text near the top (search-firm partners gate
  on these in ~20 sec before a human opens the file).
- Two pages is the exec norm; Alex's is currently 3 — **do not try to "fix" length; preserve it.**
- Tailor by narrative alignment, not keyword-chasing.

## Aakash Gupta, distilled
No standalone JD-tailoring article exists; his doctrine is reconstructed from his PM-resume corpus
(flagship guide paywalled). Transferable points:
- **X-Y-Z bullets** — lead with the accomplishment, attach a metric, name the how. "Each bullet has
  to scream result."
- **Master resume as a "component library"** — per application, pull the bullets/keywords that
  *mirror the JD's specific language*. "Using the same resume everywhere is a path to the rejection pile."
- **Cover the JD's needs** with real, quantified achievement statements — not a keyword dump.
- **Strong verbs + scale**; kill "managed"/"assisted".
- Caveat: his examples skew IC/early-career — apply the *principles*, not the example bullets, or
  you under-position a CPO.

## Self-scoring rubric (1-10 per dimension)
Gate: **any dimension < 9 → revise before delivering. Honesty (E) < 10 → hard block.** (Tightened
from the research's 7-floor because Alex's bar is a 10/10 on every criterion.)

**A. JD-alignment** — does it mirror what matters to this role?
- 9-10: **every item on the safe-levers checklist was taken or has a concrete, defensible reason to
  skip**, AND the role's core mandate sits in the scan path (headline / lead skills / lead tags /
  summary lead clauses / first experience bullet). For a near-canonical match this can be just a few
  reorders — minimal-but-complete is a 9-10, restraint is NOT penalized. For a partial match, the
  genuinely-applicable JD terms are mirrored (~75-80%) in Alex's own words where safe.
- 7-8: tailoring is honest and safe but **left an obvious zero-risk lever on the table** — e.g. the
  role's title is buried in the headline when a free reorder would surface it, or a JD-named
  requirement Alex genuinely meets is absent when one length-neutral word-swap would add it, or the
  mandate is buried below the fold and no length-neutral summary re-point was even attempted.
- 5-6: only cosmetic reordering; mandate not surfaced; multiple safe levers unused.
- 1-2: untouched; no alignment.

**B. Authenticity / no over-tailoring** — does NOT read as gamed?
- 9-10: zero near-verbatim echo; no keyword lists/stuffing; no forced phrasing; coherent narrative
  a smart reviewer trusts.
- 5-6: one or two phrasings lifted too literally, or a slightly stuffed line.
- 1-2: JD echoed verbatim; keyword salad; obvious AI-sameness.

**C. Voice / differentiation preserved**
- 9-10: Alex's leadership through-line and natural voice fully intact; non-uniform bullet rhythm;
  specific named systems/outcomes; recognizably the same person as the base.
- 5-6: voice partly sanded; through-line muddied by over-fitting.
- 1-2: generic JD-matched resume; voice erased.

**D. Layout integrity** (mechanically verified)
- 9-10: same page count as base; **DIFF_PX ≈ 0 on every unedited page and confined to the edited
  region on edited pages** (with isolated render profiles this gate is exact); no reflow,
  widows/orphans, or broken columns; every swapped element length-neutral; AND the post-edit XML diff
  shows ONLY the intended runs changed (no whole-file re-serialization / smart-quote re-encoding of
  untouched text — that blinds the diff gate even when it renders identically).
- 5-6: holds but a line renders a hair long / spacing drift not yet checked at page edges, or
  unedited runs were silently re-encoded.
- 1-2: page count changed, a section reflowed, or a stranded line appeared.

**E. Honesty** (hard block — must be 10)
- 10: every line passes the cold-defense test; every metric/scope real and owned; no inflated
  proficiency; no unprovable skill.
- ≤9: any claim needing invented context, inflated scope/metric, or unprovable skill → **block & fix.**
