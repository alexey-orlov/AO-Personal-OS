## Topics covered
- Oracle + NVIDIA + SoftServe POC pipeline: how clients get sourced and funded.
- Funding model: NVIDIA (and possibly Oracle) pay for the initial POC; client pays only if it scales.
- Active/known client cases: Bosch (Speaker A), Riyadh Air (Dmytro Duchenko), Bin Laden Group (dropped from QOpt path), DHL (early stage).
- Bosch POC deep-dive: dispatcher tool for assigning field technicians to zones using Oracle Cloud + NVIDIA cuOpt.
- Scope boundary: cuOpt handles assignment; OFSR (Oracle) handles intra-zone routing (out of scope).
- Architecture: custom UI + custom backend wrapper + cuOpt as black-box optimizer; Oracle's pre-existing UI/cuOpt setup (built for trucking) was rejected as unfit.
- Bosch workflow constraints: stable baseline schedules, temporary substitutions for sick leave/vacation, locked visits (pre-ordered parts, committed time slots) — none handled natively by cuOpt, all built on top.
- Iterative cuOpt invocation pattern (re-run per sub-period, backend stitches results).
- Timeline & team: started 2026-04-06, ~2 weeks discovery + workshops, then build. Team: 2 backend (full), 1 frontend (variable, 0.25–0.5, dropped off near end), TL ~0.5→0.25, PM/BA (Speaker A) ~0.5→0.25, Oracle-side DevOps.
- "Packaging"/productization conversations happened between Oracle and Olya Hanushchak (FE) + Vlad Selyotkin (architect); Speaker A was not invited and doesn't know outcomes.
- Speaker A's view on productization feasibility: a universal product is unrealistic; standardizing the *engagement process* (fixed-scope cuOpt POC packages) is the realistic play.
- Why not "vibecode" it in days: needed to be a credible POC on real client data with a polished UI, not a throwaway prototype.

## Decisions made
- Build Bosch POC fully from scratch (UI + backend) -> Oracle's existing trucking UI/cuOpt config was unsalvageable; rewriting cost > greenfield cost within the time/budget box.
- Keep routing (OFSR) out of POC scope -> Bosch's problem was zone-level assignment, not intra-zone sequencing.
- Build post-POC backlog items (integrations, extra logic) opportunistically during the POC -> no client/Oracle commitment yet for phase 2; done on spec to be ready if Bosch greenlights.
- Speaker B will also talk to Dmytro Duchenko (on vacation) and Olya Hanushchak -> they hold the technical detail and the Oracle "packaging" conversation context that Speaker A lacks.
