# Agentic architecture options comparison — research brief

_question: Comparing agentic-model architecture options — model-boxed harness (Claude Code style, everything inside one model) vs dedicated harness (Vercel AI SDK + Cloudflare Workers) vs enterprise platform stacks (NVIDIA AI-Q + OCI, Langfuse-based observability/orchestration, AWS/Azure native agent solutions) — tradeoffs for architecture decisions in agentic products_
_date: 2026-07-01 · sources: 8 · provenance: [drop](../../../_inbox/processed/tg-20260630-135928-172.md)_

## TL;DR
These aren't competing "architectures" for the same job — they sit on a build-vs-buy spectrum with a genuinely orthogonal observability layer. Claude Code is the model-boxed pole (thin deterministic loop, model does the reasoning); Vercel AI SDK + Cloudflare Workers is a dedicated harness you assemble from primitives; NVIDIA AI-Q + OCI and AWS Bedrock AgentCore / Azure AI Foundry are managed enterprise platforms (buy); Langfuse is a tracing/eval layer that sits over any of the other three, not an alternative to them. The real decision axis is control vs time-to-value and expected volume/scale — not which is "more advanced."

## What's known

1. **Claude Code = model-boxed harness.** The agent loop is: read state → ask the model → execute tool calls it requests → repeat. Only 1.6% of the codebase is "AI decision logic" — the rest (permissions, context compaction, tool routing, recovery) is deterministic infrastructure around that loop. Extensibility comes via MCP (tools) and Skills (procedural knowledge), not a bespoke orchestration framework. [Fact/source — Claude Code docs, code.claude.com; "Dive into Claude Code" arXiv paper 2604.14228, 2026]

2. **Vercel AI SDK + Cloudflare Workers = assembled dedicated harness.** Vercel's stack (AI SDK 6, shipped Dec 2025, + Workflow SDK + AI Gateway) targets agentic SaaS on Next.js. Cloudflare's answer is the Agents SDK + Durable Objects (state) + Workers AI (edge inference); it interoperates with Vercel's AI SDK via the `workers-ai-provider`. Named gap: Cloudflare has no equivalent to Vercel's Workflow SDK for durable multi-step agents. [Fact/source — Cloudflare Workers AI docs, developers.cloudflare.com]

3. **"Harness vs framework" is the industry's own name for this spectrum.** 2026 framing: a *framework* (LangGraph, Microsoft Agent Framework) is code you write to define agent logic; a *harness* (Claude Code, Cloudflare Agents SDK) is something you configure, with guardrails/observability already built in. Microsoft's own positioning treats the harness as a "flex point, not lock-in" — investments in LangGraph/Claude Agent SDK carry forward into Microsoft Foundry. [Practitioner consensus — MindStudio, Atlan, Microsoft Agent Framework devblog, 2026; vendor-adjacent framing, not independently audited]

4. **NVIDIA AI-Q Blueprint on OCI = the reference-architecture pole.** Open-source (April 2025), built on LangChain Deep Agents + NeMo Agent Toolkit; multi-agent with an intent router splitting queries to a Shallow Research Agent vs a Deep Research Agent. Deploys on OCI via Terraform + OKE in ~20–25 minutes; every layer (models, tools, RAG backends, sub-agents, evaluators) is swappable through YAML config or the NeMo Agent Toolkit plugin system. This is an architecture Alex/SoftServe would adapt and own, not a single SaaS control plane. [Fact/source — NVIDIA Developer Blog, developer.nvidia.com]

5. **AWS Bedrock AgentCore and Azure AI Foundry Agent Service are the two dominant managed "buy" platforms**, both reaching enterprise GA between Oct 2025 and Q1 2026. AgentCore (GA Oct 13, 2025) is the more model/framework-agnostic of the two — any framework, 200+ models via Bedrock — with first-class agent identity (OAuth + credential vault) and the most complete memory offering (managed + self-managed strategies, episodic memory at GA). Azure Foundry is tilted toward OpenAI models but wins on Microsoft 365/Teams integration (10,000+ customers at GA); its own agent memory is still Public Preview in Q1 2026, not GA. [Fact/source, cross-referenced across independent vendor-comparison write-ups — agentmarketcap.ai, bitslovers.com, callsphere.ai]

6. **Langfuse is orthogonal, not a fourth architecture choice.** Open-source (MIT), OTLP-native observability/eval platform — traces, prompt management, datasets — that instruments whatever agent loop runs underneath (model-boxed, custom harness, or managed platform). Its trace model favors prompt-centric apps; long agent runs with deeply nested tool calls and sub-agents are reported as slower to read in its UI than in agent-native alternatives. [Fact/source — Langfuse GitHub/blog; Practitioner consensus on the UI tradeoff — Laminar, aimultiple comparison pieces, 2026]

7. **Build-vs-buy economics: directionally consistent, but the specific numbers are unverified.** Several 2026 content pieces (digitalapplied, augmentcode, mindstudio, findskill) converge on "buy under roughly 100K–1M conversations/year, build above it," with build economics resting on Claude Sonnet per-token pricing undercutting per-conversation platform pricing (e.g. Salesforce Agentforce cited at $2/conversation). ⚠ These are SEO/content-marketing sources with suspiciously precise, uncited stats ("34.4% business AI adoption," "$0.54/conversation") appearing near-identically across multiple unrelated searches — treat the *crossover-volume* shape as directional only, not the specific figures. [Speculation / low-tier source — needs verification against primary Anthropic/AWS/Microsoft pricing pages]
