# AI PM Conga AI session

_source: 20260520 175738-DB70FD14.m4a_

## TL;DR
- SoftServe's Alex Orlov delivered an enablement session on AI-powered product management workflows, covering context engineering, agentic system design, and PM-specific use cases.
- Core thesis: 60% of agentic AI success depends on context engineering (knowledge, metadata, registries); model and prompt choice matter much less than commonly thought.
- Live demo showed an agent in VS Code drafting a PRD using ~80 PM skills and sub-agent reviewers (engineering, legal, UX, executive, skeptic, customer voice).

## Key points
- **Audience poll**: Most attendees reported "ad hoc" AI adoption (option A); a sales attendee noted heavy individual AI use for research/proposals/decks but no harmonization, governance, or cross-functional sharing with PM org.
- **Two agentic archetypes**: structured/no-code (e.g., Copilot Studio) vs. Git-like file repository archetype (e.g., VS Code + agent mode). Session focused on the repository archetype.
- **Building blocks introduced**:
  - *Instructions*: system prompt (~500-line root), skills (named, reusable prompt files for specific tasks — e.g., PRD drafting, competitor analysis), sub-agents used as reviewer personas.
  - *Knowledge/context layers*: project-level, user-level, conversation-level. Sources include business info, strategy/OKRs, stakeholder metadata, templates, writing style, feature index, data catalog, metrics catalog, personal context.
  - *Self-learning mechanism*: agent updates stored skills/context after interactions based on user feedback.
  - *Tools/integrations*: MCP protocol to connect to Jira, Figma, Confluence, Power BI, Teams, SharePoint, etc.
  - *Triggers*: manual, scheduled, or event-driven from external systems.
- **PM use cases walked through**:
  1. PRD drafting + multi-reviewer sub-agent critique (live demo run).
  2. Prototyping using Figma design tokens for product-accurate prototypes (GitHub Spark mentioned).
  3. Release notes auto-generated on Jira release status change.
  4. Customer insight triage — AI decides if a case is minor enough to auto-draft a ticket or even a PR fix (referenced Cursor/OpenAI internal practice).
  5. Product analytics (e.g., monthly activation analysis) via Power BI MCP integration with templated SQL queries.
- **Skill file format**: Markdown with required convention (name, description, auto-use flag, slash-command flag); body is prompt-engineering best practice.
- **Disclaimers/common mistakes**: hallucinations risk, garbage-in/garbage-out, autonomy trap, treating first-pass output as final, neglecting iteration on skill content.
- Session ran ~5 minutes over the scheduled hour; live PRD review by sub-agents could not be completed in time.
- Context: session is part of enablement for an upcoming hackathon; registrations more than doubled in the prior 24 hours (from ~81 at session start); registrations closing the same day.
